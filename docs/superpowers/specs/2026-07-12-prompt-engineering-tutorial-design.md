# 提示词工程系统教程设计说明

## 目标

在 `notes` 目录新增一份面向开发者的中文提示词工程教程。教程聚焦提示词设计、调试、评测和生产维护，不展开 RAG、Function Calling 或 Agent 的实现细节。

## 读者与前置知识

- 读者具备基础编程经验，了解大模型、token 和上下文窗口的基本含义。
- 读者希望将提示词用于真实应用，而不是只收集可复制的模板。

## 产物

新增 `notes/提示词工程系统教程.md`，采用现有学习笔记的 Markdown 风格：先解释概念，再给出对比示例、可复用模板、易错点和小结。

## 章节

1. 提示词工程解决的问题与能力边界。
2. 模型如何读取消息，以及系统、开发者、用户指令的职责。
3. 高质量提示词的基本结构：目标、上下文、约束、输出格式和验收标准。
4. 指令、上下文、约束和格式的写法与反例。
5. Few-shot 示例的作用、选择标准和常见误用。
6. 复杂任务的拆解与分阶段提示。
7. 结构化输出与 JSON 约束，不包含工具调用实现。
8. 多轮对话与上下文管理。
9. 常见失败模式与提示词调试流程。
10. 提示词评测、版本管理和回归测试。
11. 提示词注入、数据边界和安全设计。
12. 生产发布前检查清单。
13. 可复用模板与练习题。

## 质量标准

- 全文使用中文，术语第一次出现时给出简明解释。
- 每个关键章节至少包含一个“差的提示词 / 改进后提示词”对比。
- 区分提示词能解决的问题与需要系统能力解决的问题。
- 提供可执行的检查表和练习，而不是只给原则。
- 不假设读者已经掌握 RAG、Function Calling 或 Agent。

## 非目标

- 不实现或讲解向量数据库、检索增强生成、工具调用协议、Agent 编排。
- 不绑定特定模型供应商或 SDK。
