# 工具定义与参数校验速查表

## 工具定义的四个重点

```text
name：稳定、清晰、唯一
description：说明什么时候使用、什么时候不要使用
parameters：类型、必填字段、枚举值、格式和额外字段
返回结构：success、data、error_code、message、retryable
```

## 两层校验

```text
模型前：JSON Schema 帮助模型生成正确参数
业务执行前：程序再次校验权限、范围、状态和业务规则
```

Schema 不是安全边界。模型生成的参数始终是不可信输入，不能直接拼接 SQL 或直接执行写操作。

## 查询销售额示例

```json
{
  "type": "object",
  "properties": {
    "month": {"type": "string", "description": "YYYY-MM"},
    "region": {"type": "string", "enum": ["华东", "华南", "华北"]}
  },
  "required": ["month", "region"],
  "additionalProperties": false
}
```

## 常见失败

- 缺少必填参数：拒绝执行或要求模型补全
- 类型错误：返回 `INVALID_ARGUMENTS`
- 枚举值不合法：拒绝执行
- 日期格式错误：拒绝执行
- 用户无权限：返回 `PERMISSION_DENIED`，不可重试
- 合法格式但没有数据：返回 `NO_DATA`，不是系统异常
