# 业务幂等速查表

## 两种幂等

```text
请求幂等：同一个请求重试，不重复执行
业务幂等：业务状态不允许重复或超额执行
```

## 退款判断

```text
相同 key + 相同参数 -> 返回第一次结果
相同 key + 不同参数 -> IDEMPOTENCY_CONFLICT
不同 key + 累计金额超订单金额 -> REFUND_LIMIT_EXCEEDED
不同 key + 合法的分次退款 -> 可以执行，但必须重新校验余额
```

## 生产实现

- 幂等键建立唯一索引
- 订单状态和累计退款金额放在数据库
- 校验和扣减在同一个事务中完成
- 外部支付超时先查询执行状态
- 记录退款单号、请求号和审计日志
