# 工具错误与重试速查表

## 错误分类

| 错误 | 是否通常重试 | 正确动作 |
|---|---:|---|
| 网络超时、临时不可用 | 是 | 限次数重试，使用退避 |
| 参数格式错误 | 否 | 修正参数或询问用户 |
| 无权限 | 否 | 停止并说明权限不足 |
| 查询无数据 | 否 | 告知没有符合条件的数据 |
| 服务限流 | 视情况 | 延迟、退避或降级 |
| 写操作超时 | 不能盲目重试 | 先查询执行状态，再决定 |

## 工具返回建议

```json
{
  "success": false,
  "error_code": "PERMISSION_DENIED",
  "message": "当前用户无权查询该订单",
  "retryable": false,
  "request_id": "req_123"
}
```

模型需要知道发生了什么、能否重试和下一步有什么选择；最终的重试次数、权限和安全边界由应用程序控制。

## 循环保护

- 单个工具最大调用次数
- Agent 最大步骤数
- 总任务超时
- Token 或费用预算
- 相同工具和相同参数的重复调用检测
- 写操作的幂等键

## 决策顺序

```text
收到工具结果
-> 判断错误类型
-> 判断 retryable
-> 检查次数、预算和超时
-> 重试 / 修正 / 停止 / 降级 / 转人工
```
