# 写操作 Tool 安全速查表

## 执行顺序

```text
解析模型参数
-> 业务参数校验
-> 用户和租户权限校验
-> 查询幂等键
-> 检查是否需要人工审批
-> 执行真实写操作
-> 保存结果和审计记录
```

## 不应该由模型决定的内容

- 当前用户是否有权限
- 是否通过人工审批
- 可操作的租户和数据范围
- 幂等记录是否已经成功
- 是否允许绕过金额限制

## 幂等结果

```text
相同 key + 相同请求：返回第一次结果
相同 key + 不同请求：IDEMPOTENCY_CONFLICT
外部调用超时：先查询执行状态，不能盲目重试
```

## Human-in-the-loop

模型可以提出“需要退款”，但订单号需要由业务层校验，幂等键应由请求入口或业务服务生成。审批状态应来自业务审批系统，通过执行上下文传入工具，不能使用模型生成的 `approved=true`。
