# Tool Calling 速查表

## 核心流程

```text
用户问题
  -> 第一次模型请求：消息 + 工具定义
  -> 模型返回 tool_call
  -> 应用程序解析参数并校验
  -> 应用程序执行真实工具
  -> 追加 assistant 消息和 tool 消息
  -> 第二次模型请求
  -> 最终回答
```

## 责任边界

| 组件 | 负责什么 | 不负责什么 |
|---|---|---|
| 模型 | 理解问题、选择工具、生成参数、组织答案 | 直接执行数据库或外部 API |
| 应用程序 | 解析、校验、执行工具、维护消息状态 | 把模型输出当作天然可信 |
| 工具函数 | 执行业务查询或动作并返回结构化结果 | 自己决定用户是否有权限 |

## 真实测试与单元测试

```text
单元测试：FakeClient，快速、稳定、不消耗额度
真实测试：读取 .env，调用真实模型，验证端到端链路
```

真实测试应显式开启，避免普通测试意外产生 API 费用。
