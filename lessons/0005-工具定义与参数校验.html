工具定义与参数校验

模型能不能正确调用工具，很大程度取决于工具定义；系统安不安全，则取决于业务代码是否再次校验。

工具定义不是装饰

工具定义是模型做决策时看到的接口说明。名称帮助模型定位能力，描述告诉模型什么时候使用，参数 Schema 约束模型应该生成什么结构。

SALES_TOOL = {
    "type": "function",
    "function": {
        "name": "query_sales",
        "description": "按月份和地区查询销售额，只执行只读查询。",
        "parameters": {
            "type": "object",
            "properties": {
                "month": {"type": "string"},
                "region": {
                    "type": "string",
                    "enum": ["华东", "华南", "华北"],
                },
            },
            "required": ["month", "region"],
            "additionalProperties": False,
        },
    },
}

为什么要两层校验

Schema 主要帮助模型生成符合格式的参数，但它不是安全边界。攻击者可以绕过模型直接构造请求，模型也可能生成符合 Schema 但不符合业务规则的值。

第一层：Schema
{"month": "2026-06", "region": "华东"}

第二层：业务函数
month 是否符合 YYYY-MM？
region 是否在允许范围？
当前用户是否有权限？
查询范围是否允许？
原则：模型输出始终是不可信输入。Schema 负责帮助模型，业务代码负责保护系统。

项目中的业务校验

02 练习的 execute_tool 先判断工具名称，再把参数交给 query_sales。销售查询函数继续检查月份格式、地区是否存在以及是否有对应数据。

def query_sales(month: str, region: str) -> dict[str, object]:
    if not MONTH_PATTERN.fullmatch(month):
        return {
            "success": False,
            "error_code": "INVALID_MONTH",
            "retryable": False,
        }

    if region not in VALID_REGIONS:
        return {
            "success": False,
            "error_code": "UNKNOWN_REGION",
            "retryable": False,
        }

    # 格式合法，但没有数据，是 NO_DATA 而不是系统故障
    return find_sales(month, region)

错误类型要表达清楚

情况错误码是否重试
缺少或格式错误INVALID_ARGUMENTS修正或询问
地区不支持UNKNOWN_REGION否
没有符合条件的数据NO_DATA否
无权限PERMISSION_DENIED否
