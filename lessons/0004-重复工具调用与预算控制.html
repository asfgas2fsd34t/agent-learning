重复工具调用与预算控制

最大步数保护整个任务，重复调用检测保护某一个工具。两者配合，才能避免 Agent 在错误状态里持续浪费时间和额度。

为什么只限制步骤还不够

max_steps=5 能保证任务最多运行五轮，但它不能说明五轮是否都在做有价值的事情。如果模型五次都用同一个工具、传入同样参数，Agent 只是重复消耗请求。

第 1 轮：query_sales(2026-06, 华东)
第 2 轮：query_sales(2026-06, 华东)
第 3 轮：query_sales(2026-06, 华东)
第 4 轮：query_sales(2026-06, 华东)
第 5 轮：query_sales(2026-06, 华东)

如何定义“相同调用”

不能只比较工具名。下面两个调用使用的是同一个工具，但参数不同，不能判定为重复：

query_sales({"month": "2026-06", "region": "华东"})
query_sales({"month": "2026-06", "region": "华南"})

项目中使用“工具名 + 排序后的 JSON 参数”构造调用签名：

signature = (
    f"{tool_call.function.name}:"
    f"{json.dumps(arguments, sort_keys=True, ensure_ascii=False)}"
)
call_counts[signature] = call_counts.get(signature, 0) + 1

达到上限后怎么做

超过 max_same_call 后，程序不再执行真实工具，而是构造一个结构化错误，并把它作为 tool 消息返回给模型：

tool_result = {
    "success": False,
    "error_code": "DUPLICATE_TOOL_CALL",
    "message": "相同工具和参数已达到重复调用上限",
    "retryable": False,
}

这样模型能看到失败原因并结束或改变策略。但注意：这只是局部保护，外层仍然需要 max_steps，因为模型可能换一个参数继续循环。

三层预算保护

局部：max_same_call，限制同一工具和相同参数。

任务：max_steps，限制整个 Agent 的执行轮数。

系统：总超时、Token 预算和费用预算，限制任务的总体资源消耗。

生产环境还需要记录每次调用的工具名、参数摘要、耗时、Token、错误码和 request_id，这样才能解释 Agent 为什么停止或重试。
