Agent 循环与状态

前两课是一次 Tool Calling；这一课把“模型、工具、结果”放进一个受保护的循环，让 Agent 能根据反馈继续行动。

从一次调用到循环

你现在的 02 练习最多执行这一条链路：

模型 -> 工具 -> 模型 -> 结束

真正的 Agent 允许第二次模型请求继续提出新的工具调用：

模型
  -> 工具 A
  -> 观察结果
  -> 模型决定下一步
  -> 工具 B
  -> 观察结果
  -> 模型输出最终答案

最小可用代码

核心不是“让模型无限执行”，而是每一轮都把新的 assistant 消息和 tool 消息追加到 messages，让下一轮模型能看到完整状态。

MAX_STEPS = 5

def run_agent(question, settings, client):
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": question},
    ]

    for step in range(MAX_STEPS):
        response = client.chat.completions.create(
            model=settings.model,
            messages=messages,
            tools=TOOLS,
            tool_choice="auto",
        )
        assistant = response.choices[0].message

        if not assistant.tool_calls:
            if not assistant.content:
                raise ValueError("模型没有生成最终回答")
            return assistant.content

        messages.append({
            "role": "assistant",
            "content": assistant.content,
            "tool_calls": serialize_tool_calls(assistant.tool_calls),
        })

        for tool_call in assistant.tool_calls:
            result = execute_tool_call(tool_call)
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": json.dumps(result, ensure_ascii=False),
            })

    return "执行步骤超过上限，已停止。"

这段代码中的状态

messages 是最小的短期记忆。它记录用户目标、模型之前做出的工具选择和工具返回结果。没有这些消息，下一轮模型就不知道已经查过什么，也不知道上一步失败的原因。

messages = [
    {"role": "system", "content": "你是销售助手"},
    {"role": "user", "content": "分析华东区销售额"},
    {"role": "assistant", "tool_calls": [...]},
    {"role": "tool", "tool_call_id": "call_1", "content": "..."},
]
注意：模型的“思考”不是可靠的状态。可靠状态应该由程序明确保存，例如已执行工具、工具参数、执行次数和结果。

为什么必须有最大步数

模型可能因为工具结果不清晰而反复调用同一个工具。MAX_STEPS 是最后一道保险：无论模型怎么判断，程序最多只允许执行指定轮数。

生产系统还应继续增加单工具次数、总超时、Token 预算和重复参数检测。达到限制后，可以返回明确失败、降级到人工处理，或者保存任务状态等待继续。

和 02 练习的对应关系

02 练习里的 first_response 和 final_response 已经是循环的第 1 轮和第 2 轮，只是代码把循环次数写死为一次工具调用。下一步改造时，应该把“第二次请求”放入 for 循环，并在每轮结束时判断是否有新的 tool_calls。

当前练习：第一次请求 -> 执行工具 -> 第二次请求 -> 结束
循环版本：请求 -> 工具 -> 结果 -> 再请求 ... -> 最终回答或达到上限
