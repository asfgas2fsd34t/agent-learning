# Transformer 基础笔记

这份笔记整理了关于 Transformer 的一组基础概念，重点包括：

- CNN、RNN、Transformer 的思路区别
- Transformer 的核心创新
- 编码器和解码器分别做什么
- Q、K、V 到底是什么
- 前馈网络在干什么

目标不是一次讲到特别深，而是先把主干概念和最关键的公式串起来。

---

## 1. CNN、RNN、Transformer 分别是什么逻辑

### 1.1 CNN 的逻辑

CNN 的核心思路是：

**先看局部，再逐层组合成整体理解。**

它特别适合图像，因为图像最重要的特征通常先出现在局部区域里，比如：

- 边缘
- 纹理
- 角点
- 局部形状

卷积核会在图像上滑动，用同一套参数反复提取局部特征。这叫参数共享。

CNN 的优势：

- 参数量比全连接小
- 擅长局部模式提取
- 对图像任务很自然

CNN 的局限：

- 更擅长局部，不天然擅长全局关系
- 想看到远距离依赖，通常要堆很多层
- 对文本和时间序列不够自然

### 1.2 RNN 的逻辑

RNN 的核心思路是：

**按顺序一个一个处理，并把前面的状态传给后面。**

它很适合：

- 文本
- 语音
- 时间序列

RNN 的优势：

- 能表达顺序
- 天然适合“前文影响后文”

RNN 的局限：

- 长距离依赖难学
- 梯度消失 / 梯度爆炸
- 很难并行
- 序列越长越慢

### 1.3 Transformer 的逻辑

Transformer 的核心思路是：

**不再让信息按时间一步步传，而是让每个位置直接去看所有位置。**

也就是说：

- CNN 偏局部
- RNN 偏顺序传递
- Transformer 直接全局建模

它最重要的收益有两个：

- 更容易处理长距离依赖
- 更适合并行训练

---

## 2. Transformer 的重点创新

Transformer 最值得记住的创新点有 4 个。

### 2.1 Self-Attention

每个 token 在计算自己的新表示时，不只是看自己，而是会去看整句里所有 token。

这让模型可以直接建立远距离关系。

### 2.2 不再依赖顺序递推

RNN 是一个时间步接一个时间步算。Transformer 不是。

Transformer 可以一次把整段输入送进去，通过矩阵运算并行完成。

### 2.3 Multi-Head Attention

不是只算一套注意力，而是同时算多套。

这样不同的头可以学不同类型的关系，比如：

- 语法关系
- 代词指代
- 长距离依赖
- 局部搭配

### 2.4 位置编码

因为 Transformer 自身不带顺序感，所以要显式加入位置编码，告诉模型：

- 谁在前
- 谁在后
- 谁离谁更近

---

## 3. 输入、维度和矩阵

假设一句话一共有 `n` 个 token，每个 token 的表示维度是 `d_model`。

那么整句话可以写成一个矩阵：

\[
X \in \mathbb{R}^{n \times d_{model}}
\]

其中：

- 一行表示一个 token
- 一列表示这个 token 的某个特征

如果加入位置编码矩阵 `P`，那么输入会变成：

\[
X' = X + P
\]

这表示：

- `X` 是 token embedding
- `P` 是位置编码
- `X'` 是真正进入 Transformer 的输入

---

## 4. Q、K、V 是什么

Q、K、V 是 attention 的三个核心向量：

- `Q` = Query
- `K` = Key
- `V` = Value

最直白的理解是：

- `Q`：我现在想找什么信息
- `K`：我这里有什么信息可供匹配
- `V`：如果你关注我，你最终拿走什么内容

它们来自输入矩阵的三次线性变换：

\[
Q = X'W^Q
\]

\[
K = X'W^K
\]

\[
V = X'W^V
\]

如果：

- `X'` 是 `n × d_model`
- `W^Q` 是 `d_model × d_k`
- `W^K` 是 `d_model × d_k`
- `W^V` 是 `d_model × d_v`

那么：

- `Q` 是 `n × d_k`
- `K` 是 `n × d_k`
- `V` 是 `n × d_v`

---

## 5. `QK^T` 到底是什么意思

\[
QK^T
\]

表示：

**每个 token 的 query，去和所有 token 的 key 做点积，得到相关性分数矩阵。**

如果：

- `Q` 是 `n × d_k`
- `K^T` 是 `d_k × n`

那么：

\[
QK^T \in \mathbb{R}^{n \times n}
\]

这个 `n × n` 矩阵的含义是：

- 第 `i` 行第 `j` 列表示
- 第 `i` 个 token 对第 `j` 个 token 的关注分数

矩阵里的一个元素可以写成：

\[
S_{ij} = q_i \cdot k_j
\]

意思是：

- 第 `i` 个 token 的 query
- 和第 `j` 个 token 的 key
- 做点积得到相似度

---

## 6. Attention 的核心公式

Transformer 最核心的公式是：

\[
Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
\]

可以拆成 4 步。

### 第 1 步：算分数

\[
QK^T
\]

得到每个 token 对所有 token 的相关性分数。

### 第 2 步：缩放

\[
\frac{QK^T}{\sqrt{d_k}}
\]

这是为了让数值更稳定，避免维度大时点积太大。

### 第 3 步：softmax

\[
A = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)
\]

得到注意力权重矩阵 `A`。

每一行加起来等于 1，表示：

**当前 token 把多少注意力分给每个 token。**

### 第 4 步：对 V 加权求和

\[
O = AV
\]

得到新的输出表示 `O`。

这一步的含义是：

**按注意力权重，从所有 token 的 value 里取信息，再加权整合。**

---

## 7. Multi-Head Attention

Transformer 不只做一次 attention，而是做很多次。

第 `i` 个头的公式是：

\[
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
\]

然后把所有头拼起来：

\[
MultiHead(Q,K,V)=Concat(head_1,\dots,head_h)W^O
\]

这里的直觉是：

- 一个头学一种关系不够
- 多个头可以从多个角度同时理解上下文

这就是多头注意力。

---

## 8. 前馈网络 FFN 是干什么的

前馈网络 `Feed Forward Network` 的作用可以记成一句话：

**attention 负责“词和词交流”，FFN 负责“每个词自己再加工一遍”。**

论文里的公式写作：

\[
FFN(x)=max(0, xW_1+b_1)W_2+b_2
\]

也就是：

1. 线性层
2. 激活函数 ReLU
3. 线性层

在 Transformer 里，FFN 是对每个 token 分别独立执行的，不负责跨 token 通信。

它最主要的作用是：

- 增加非线性表达能力
- 把 attention 融合后的信息再加工一遍

## 9. 编码器和解码器分别做什么

原始 Transformer 是 `Encoder-Decoder` 结构。

### 9.1 编码器 Encoder 的作用

编码器的任务是：

**把输入句子变成一组带上下文的语义表示。**

比如输入：

`I love machine learning`

编码器处理后，每个词的表示都已经融合了整句上下文。

### 9.2 解码器 Decoder 的作用

解码器的任务是：

**一边看已经生成的内容，一边看编码器输出，逐步生成下一个词。**

比如翻译成：

`我 爱 机器学习`

它不是一次性全生成，而是：

1. 先生成 `我`
2. 再生成 `爱`
3. 再生成 `机器学习`
4. 最后生成结束符

### 9.3 Decoder 为什么要 Mask

因为生成时不能偷看未来，所以 decoder 的 self-attention 要做 mask：

- 当前词只能看左边
- 不能看右边未来位置

---

## 10. 为什么后来有的模型只用编码器，有的只用解码器

### 10.1 Encoder-only

代表模型：`BERT`

特点：

- 双向看上下文
- 更适合理解任务

擅长：

- 分类
- 检索
- 匹配
- 抽取

### 10.2 Decoder-only

代表模型：`GPT`

特点：

- 单向建模
- 只能看左边
- 更适合生成任务

擅长：

- 对话
- 续写
- 代码生成
- 长文本输出

### 10.3 为什么 GPT 也能“理解”

因为 GPT 虽然是 decoder-only，但在开始回答之前，你的整段输入已经都在它左边了。

所以它完全可以通过 masked self-attention 读取你整段问题，再基于这个上下文生成回答。

它不是“先理解、再生成”两步分开，而是：

**为了生成下一个 token，必须理解前面的上下文。**

---


## 14. 一组最值得背的结论

### 关于 CNN、RNN、Transformer

- CNN 擅长局部模式
- RNN 擅长顺序传递
- Transformer 擅长全局关系建模

### 关于 Transformer 核心创新

- 用 self-attention 替代顺序递推
- 解决长距离依赖问题
- 支持并行计算
- 用多头注意力增强表达能力

### 关于 QKV

- Q：我想找什么
- K：我这里有什么可匹配的信息
- V：如果你关注我，你拿走什么内容

### 关于 FFN

- attention 负责交流
- FFN 负责每个 token 自己再加工
---

## 15. 一句话总总结

Transformer 的本质是：

**先把输入变成 token 向量，再通过 Q、K、V 计算每个位置该关注谁，用注意力机制整合全局上下文，最后通过多头注意力和前馈网络不断加工表示，从而实现高效的理解与生成。**
