# 06 生产级工具集成

## 学习目标

这一课把搜索、SQL、文件、HTTP API 和自定义业务函数接入统一工具协议。重点是边界和安全，而不是工具数量。

## 工具分类

| 类型 | 典型用途 | 主要风险 |
| --- | --- | --- |
| 搜索 | 查公开或内部索引 | 注入、过时信息、来源不可信 |
| SQL | 查询业务数据 | 越权、注入、全表扫描 |
| 文件 | 读取知识文件 | 路径穿越、敏感文件泄露 |
| HTTP API | 调用外部系统 | 超时、重试、副作用 |
| 自定义业务工具 | 下单、退款、审批 | 权限、幂等、状态一致性 |

## 统一返回协议

```python
class ToolResult(BaseModel):
    success: bool
    data: dict | None = None
    error_code: str | None = None
    message: str
    retryable: bool = False
```

工具实现不同，但 Agent 接收到的结果结构应稳定。

## SQL 工具

- 只开放参数化查询或经过审批的查询模板。
- 只读工具使用只读数据库账号。
- 根据服务端 `user_id/tenant_id` 自动追加数据范围，不能信任模型参数。
- 限制行数、执行时间和允许访问的表。

## 文件工具

- 将目标路径解析成绝对路径。
- 验证路径仍位于允许的根目录。
- 限制扩展名和文件大小。
- 不允许模型读取 `.env`、密钥和系统目录。

## HTTP 工具

- 设置连接和读取超时。
- 仅对幂等请求有限重试。
- 写操作使用业务幂等键。
- 对响应大小和 Content-Type 做校验。
