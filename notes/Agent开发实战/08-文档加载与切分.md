# 08 文档加载与切分

## 学习目标

把 Markdown、文本等原始文件转换成带元数据的 `Document`，再切成适合检索的 chunk。这是 RAG 的数据入口。

## Document

```python
from langchain_core.documents import Document

Document(
    page_content="正文",
    metadata={"source": "guide.md", "category": "agent"},
)
```

`page_content` 是用于 Embedding 和模型上下文的正文，`metadata` 用于来源引用、权限过滤、版本和分类。

## 加载器边界

Loader 只负责可靠读取和转换，不负责回答问题。生产加载器应处理：

- 编码和文件类型
- 文件大小限制
- 重复文档与版本
- 来源、更新时间和租户元数据
- 解析失败记录
- 敏感信息过滤

## 为什么要切分

整份文档直接向量化会让不同主题混在一个向量里，也会占用大量上下文。chunk 太小会丢失语义，太大会降低检索精度。

## RecursiveCharacterTextSplitter

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=80,
)
chunks = splitter.split_documents(documents)
```

`chunk_overlap` 让相邻块保留部分上下文，但过大会增加重复、存储和 token 成本。

## Chunk 元数据

每个 chunk 至少应保留：

```text
source
document_id
version
chunk_index
tenant_id 或权限标签
```

不要在检索后才补权限；权限字段应在入库时就建立。
