# 08 文档加载与切分

## 本课的位置

RAG 的质量上限通常在生成之前就被决定：文档是否读对、结构是否保留、chunk 是否能独立表达一个事实、权限和版本是否跟着文本进入索引。本课处理的是：

```text
文件 -> Document -> 规范化 -> chunk -> 元数据 -> Embedding
```

Loader 负责读取，Splitter 负责切分，Metadata 负责追踪和过滤；它们都不负责回答问题。

## 1. Document 是什么

```python
Document(
    page_content="正文",
    metadata={
        "source": "guide.md",
        "document_id": "guide",
        "version": "2026-07-20",
        "tenant_id": "t1",
    },
)
```

`page_content` 是要被向量化和送入上下文的证据；`metadata` 是检索、权限、引用、更新和审计所需的控制信息。不要把权限标签仅写在正文里，否则向量检索和模型都可能忽略它。

## 2. 加载阶段的真实边界

生产 pipeline 通常分四层：

```text
Loader：读取 bytes，识别编码和文件类型
Parser：提取 Markdown 标题、表格、代码和正文
Normalizer：统一空白、换行、HTML 实体和敏感信息策略
Enricher：补 source、版本、租户、权限、更新时间、哈希
```

练习中的 `load_documents` 只支持 `.md/.txt`，并使用相对 `source` 和文件 stem 作为 `document_id`。它是学习接口的最小实现，不包含文件大小限制、哈希去重和权限服务；不要直接当成生产上传接口。

## 3. 为什么不能只按字符数切分

`chunk_size=200` 代表 splitter 使用的长度单位，具体实现和语言有关，不能简单等同于 200 个 token。切分时应优先保留语义边界：标题、段落、列表、代码块、表格行。`RecursiveCharacterTextSplitter` 会按一组分隔符递归尝试，但它不理解业务概念，也不会自动保证一个 chunk 自洽。

常见失败模式：

- 太大：多个主题混在一个向量中，召回噪声增加，上下文成本上升。
- 太小：定义和限制被拆开，模型只看到半个事实。
- overlap 太大：重复存储和重复召回，token 成本上升。
- overlap 太小：跨边界的条件、例外和结论丢失。

参数应使用真实问题集比较，而不是迷信固定的 500/100。先抽样查看 chunk，再测检索命中。

## 4. 元数据必须可追踪

每个 chunk 至少保留：

```text
source、document_id、version、chunk_index、content_hash
tenant_id、visibility、updated_at
```

引用时使用 `source + chunk_index`，更新时使用 `document_id + version` 删除旧版本。若来源路径会变化，应额外保存稳定的业务文档 ID。权限过滤字段必须在索引前建立，查询时由服务端身份决定过滤值。

## 5. 增量更新与删除

不要每次上传都无脑追加：

```text
读取文档 -> 计算 content_hash
  -> hash 未变化：跳过
  -> 版本变化：删除旧版本 chunk，写入新 chunk
  -> 文档删除：按 document_id 删除全部 chunk
```

“删除旧版本”是向量库的工程能力，不是 Embedding 模型的职责。没有删除能力时，至少在 metadata 中维护 active_version，并在检索时过滤；否则旧政策可能和新政策同时被召回。

对应代码：[practice/12-document-processing](../../practice/12-document-processing/README.md)

`split_documents` 先校验 `chunk_overlap < chunk_size`，再调用 splitter，最后按 `source` 生成从 0 开始的 `chunk_index`。测试验证了：只加载允许后缀、切分确实产生多个块、元数据保留且编号正确。

## 7. 可执行实验

用同一篇包含标题、列表和例外条件的 Markdown，分别运行：

```python
split_documents(documents, chunk_size=80, chunk_overlap=0)
split_documents(documents, chunk_size=80, chunk_overlap=20)
split_documents(documents, chunk_size=240, chunk_overlap=40)
```

打印每个 chunk 的 `chunk_index` 和 `page_content`，检查“条件”和“结论”是否被拆开。再修改一个文件内容，思考如何用 hash 判断是否需要重建。
