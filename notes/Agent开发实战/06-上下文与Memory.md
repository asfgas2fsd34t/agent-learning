# 05 上下文与 Memory

## 学习目标

让 Agent 在同一个会话中记住前文，同时保证不同会话隔离，并理解 Checkpointer 与上下文窗口不是同一个概念。

## 两个关键概念

```text
Checkpointer：保存 Agent 每一步状态
上下文窗口：模型一次请求最多能处理的 token
```

保存了状态不代表所有历史都应该无限传给模型。历史过长仍需裁剪或摘要。

## 使用 Checkpointer

```python
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model=model,
    tools=tools,
    checkpointer=InMemorySaver(),
)
```

每个会话使用稳定的 `thread_id`：

```python
config = {"configurable": {"thread_id": "conversation_1001"}}
agent.invoke({"messages": [...]}, config=config)
```

相同 `thread_id` 继续同一会话，不同 `thread_id` 必须隔离。

## 短期与长期记忆

短期记忆保存当前任务消息、工具结果和执行状态。长期记忆保存跨会话仍有价值的用户偏好或稳定事实。长期记忆不能每次全量注入，应按当前问题召回。

## 如何判断消息是否进入长期记忆

长期记忆没有统一的时间阈值。判断标准不是“这条消息已经过去多久”，而是：

```text
这条信息未来是否仍然有用、是否相对稳定、是否值得长期保存？
```

可以按以下流程判断：

1. **是否明确表达了事实、偏好、目标或规则？**
2. **未来任务是否可能再次使用？**
3. **信息是否相对稳定，而不是一次性的临时状态？**
4. **是否得到用户明确授权，或者符合产品的记忆策略？**
5. **是否包含密码、密钥、身份证号等不应长期保存的敏感信息？**
6. **是否已经存在相同或冲突记忆？**

只有通过筛选，才将候选记忆写入长期存储。模型可以负责提取候选记忆，但最终写入、权限和删除应由应用程序控制。

### 通常值得保存

```text
“以后经营报告中的金额统一使用万元。”
“我负责华东区的经营分析。”
“公司的销售额默认使用含税口径。”
```

这些信息具有稳定性和未来复用价值。

### 通常不值得保存

```text
“谢谢。”
“我今天有点累。”
“这次查询先看华东区。”
```

它们通常是礼貌表达、临时状态或单次任务约束。

### “历史消息”不是按时间简单划分

“历史消息”至少有三种含义：

| 类型 | 含义 | 处理方式 |
|---|---|---|
| 当前上下文 | 本次请求中直接传给模型的消息 | 受上下文窗口和 Token 预算限制 |
| 会话历史 | 当前 `thread_id` 之前的对话和任务状态 | 持久化后可摘要、裁剪或按需召回 |
| 长期记忆 | 跨会话仍有价值的事实、偏好和经验 | 独立存储，检索后注入当前上下文 |

一条消息即使是刚刚产生的，也可能是“历史消息”，只要它已经不属于当前正在处理的上下文；一条几个月前的消息，如果保存的是当前仍有效的用户偏好，也可能仍然是长期记忆。

### 推荐的时间策略

时间可以作为辅助条件，但不能作为唯一判断标准。可以采用以下示例策略：

```text
当前任务状态：任务结束或会话超时后归档
最近对话：保留最近若干轮，直到达到 Token 预算
历史摘要：对较早消息定期摘要，并保留关键决策和未完成事项
用户偏好：长期保存，但允许用户修改、删除和重新确认
临时偏好：设置较短 TTL，到期后不再召回
业务数据：不要写成记忆，实时从数据库或 BI 系统查询
```

具体保留多少轮、多少天，应根据任务类型、成本、隐私要求和合规要求配置，不存在适用于所有 Agent 的固定天数。

### BI Agent 的判断示例

```text
用户：以后报告默认使用万元。
处理：写入长期记忆。

用户：这次只看华东区。
处理：写入当前任务状态，不覆盖长期默认地区。

用户：查询 2026 年 6 月销售额。
处理：调用数据库获取实时数据，不把销售额结果当成长期记忆。

用户：上次分析发现华东销售下降主要来自订单量减少。
处理：可以保存为历史分析事件，但再次使用前要标记来源和时间，不能替代最新数据验证。
```

### 核心原则

```text
长期记忆按未来价值和稳定性筛选，不按消息年龄机械决定；
历史消息按上下文、会话和长期记忆分层管理；
实时业务数据永远以数据源为准，不能依赖过期记忆。
```

## 上下文治理

- 只保留与当前任务有关的消息。
- 工具返回只保留必要字段，避免塞入完整数据库结果。
- 长对话使用摘要，但保留关键决策、约束和未完成状态。
- 敏感信息不应因为“记忆”而永久保存。
- 用户明确说“本次”时，不应覆盖长期偏好。

## 持久化选择

`InMemorySaver` 适合学习和单进程测试，服务重启后状态丢失。生产环境应使用官方支持的数据库 Checkpointer，并设置租户隔离、保留期限和加密策略。
