# Token 和上下文窗口笔记

这份笔记只讲两个概念：

- `token` 是什么
- `上下文窗口` 是什么

这两个词在学习大模型时出现频率非常高，必须单独理解清楚。

---

## 1. token 是什么

token 是模型处理文本时使用的最小单位。

注意：

**token 不一定等于一个字，也不一定等于一个单词。**

它可能是：

- 一个汉字
- 一个完整单词
- 一个单词的一部分
- 一个标点
- 一个常见子串

也就是说，token 是 tokenizer 切出来的单位，不是我们平时自然理解的“字词”单位。

---

## 2. 为什么不直接按字或按单词处理

如果只按字处理：

- 序列会变得很长
- 表达能力不够灵活

如果只按完整单词处理：

- 词表会特别大
- 遇到生词很麻烦

所以现在大模型通常会采用 `subword` 级别的切分方法。

核心思想是：

- 常见词块尽量合并
- 生僻词拆开处理

这样能兼顾：

- 词表大小
- 文本表达能力
- 模型泛化能力

---

## 3. token 在模型里是怎么用的

比如一句话：

`解释一下 transformer`

模型不会直接处理原始文字，而是先经过 tokenizer，变成一串 token。

再把这些 token 映射成 token id，例如：

```text
[1256, 987, 44321, ...]
```

然后 token id 再进入 embedding 层，变成向量。

所以模型底层真正处理的是：

- token id
- token embedding

不是直接处理“文字本身”。

---

## 4. token 的维度是什么意思

token 一开始只是一个编号，本身没有“几维”这种说法。

只有当它进入 embedding 层，被映射成向量后，才有维度。

这个维度通常叫：

\[
d_{model}
\]

比如：

\[
d_{model} = 512
\]

意思就是：

**每个 token 会被表示成一个 512 维向量。**

所以“token 的维度”真正指的是：

**token embedding 的维度。**

---

## 5. 一个直观例子

假设句子是：

`我喜欢机器学习`

它可能会被切成：

- `我`
- `喜欢`
- `机器`
- `学习`

也可能切成：

- `我`
- `喜`
- `欢`
- `机器学习`

这取决于具体模型使用的 tokenizer。

所以：

- 同一句话，在不同模型里，token 数不一定一样
- 相同字数，不一定对应相同 token 数

---

## 6. 什么是上下文窗口

上下文窗口 `context window` 指的是：

**模型一次最多能同时处理多少个 token。**

比如一个模型上下文窗口是：

- `8k`
- `32k`
- `128k`

意思就是：

- 最多看 8000 个 token
- 最多看 32000 个 token
- 最多看 128000 个 token

它不是按“字数”算的，而是按 token 数量算的。

---

## 7. 上下文窗口里包括什么

上下文窗口里不只是你当前这一句话。

它通常包括：

- 系统提示词
- 历史对话
- 你当前输入
- 模型已经生成的回答

这些内容加起来，一起占用上下文窗口。

所以窗口大小实际上是在限制：

**这一轮推理时，模型最多能同时参考多少内容。**

---

## 8. 为什么叫“窗口”

因为模型不是无限记忆。

你可以把它理解成：

- 窗口里面的内容：模型这次能看到
- 窗口外面的内容：模型这次看不到

所以它更像“当前工作记忆”，不是永久记忆。

---

## 9. 一个对话例子

假设某个模型的上下文窗口是 `1000 token`。

当前这轮对话里：

- 系统提示占 `200`
- 历史消息占 `500`
- 你现在的问题占 `200`

那么已经用了：

```text
900 token
```

这时模型还能继续生成的大致空间只剩：

```text
100 token
```

如果继续超过，就会触发截断、报错，或者需要丢弃前面的部分上下文。

---

## 10. 为什么上下文窗口很重要

上下文窗口大小直接影响模型能做什么。

### 窗口小

- 不能直接读长文档
- 很难保留长对话历史
- 复杂任务容易丢上下文

### 窗口大

- 更适合长文总结
- 更适合代码分析
- 更适合多轮长对话
- 更适合文档问答

---

## 11. 为什么代码、表格、JSON 特别占 token

因为 tokenizer 遇到这类结构化内容时，通常会切得更碎。

比如一段代码：

```python
for i in range(10):
```

可能会被切成很多小片段：

- `for`
- ` i`
- ` in`
- ` range`
- `(`
- `10`
- `)`
- `:`

所以代码、日志、JSON、表格，往往比同长度自然语言更吃 token。

---

## 12. token 和上下文窗口的关系

它们的关系很直接：

**上下文窗口的大小，就是按 token 数量来计算的。**

所以：

- token 是“文本怎么被切”
- 上下文窗口是“切完以后最多能装多少”

---

## 13. 最容易记住的类比

你可以把它们想成：

### token

像把一段文字拆成很多小积木块。

### 上下文窗口

像一张桌子的大小。

桌子越大，一次能摆上的积木块越多。

所以：

- token 决定“拆成多少块”
- 上下文窗口决定“一次最多摆多少块”

---

## 14. 最简总结

### token

是模型处理文本时的最小单位，不一定等于一个字或一个单词。

### 上下文窗口

是模型一次最多能同时看见和利用的 token 数量上限。

---

## 15. 一句话总总结

**token 是文本进入模型前被切开的单位，上下文窗口是这些单位一次最多能放进模型“工作记忆”里的总数量。**
