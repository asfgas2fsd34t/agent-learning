# Agent 基础四模块面试题

## 覆盖范围

本题库覆盖当前已经学习的四个模块：

1. Agent 核心概念
2. Agent 架构模式
3. Tool Calling
4. Memory

题目暂不附参考答案。练习时先独立回答，再记录面试反馈和参考答案。

---

## 一、Agent 核心概念

### 1.【基础】什么是 Agent？

请用 1 分钟说明 Agent 的定义，并说出一个 Agent 系统通常包含哪些核心组成部分。

### 2.【基础】Agent 和普通大模型调用有什么区别？

请分别从目标、工具、状态和执行流程四个角度进行比较。

### 3.【基础】为什么说“大模型是 Agent 的大脑，但大模型不等于 Agent”？

请结合一个具体业务例子说明。

### 4.【场景】哪些任务不应该使用 Agent？

请举出两个不需要 Agent 的任务，并解释为什么普通代码、固定工作流或单次 LLM 调用更合适。

### 5.【场景】设计一个销售数据分析 Agent

假设用户说：“帮我分析本月销售额下降的原因。”这个 Agent 至少需要哪些能力、状态和工具？

### 6.【进阶】如何限制 Agent 的自主性？

如果 Agent 可以查询数据、发送邮件和执行退款，你会采用哪些措施避免它执行错误或危险操作？

## 二、Agent 架构模式

### 7.【基础】什么是 ReAct？

请描述 ReAct 的完整循环，并说明 `Thought`、`Action`、`Observation` 分别起什么作用。

### 8.【基础】ReAct 适合什么任务？它有哪些缺点？

至少说出两个适用场景和三个工程风险。

### 9.【基础】Plan and Execute 与 ReAct 的关键区别是什么？

请从是否具有全局计划、适用任务长度、成本和可控性几个方面比较。

### 10.【基础】什么是 Reflection？

Reflection 和普通的失败重试有什么区别？

### 11.【基础】什么时候才需要 Multi-Agent？

为什么不能遇到复杂任务就直接拆成多个 Agent？

### 12.【场景】架构模式选择

下面四个任务分别适合使用什么实现方式？请说明理由。

```text
A. 把一段用户评论分类为正面或负面
B. 根据日志和源码排查接口 500 错误
C. 生成包含多项指标和原因分析的月度经营报告
D. 生成代码、运行测试，并根据测试错误修改代码
```

### 13.【场景】组合多种架构模式

为“复杂 BI 分析助手”设计架构。请说明 ReAct、Plan and Execute、Reflection 分别放在哪一层，以及暂时是否需要 Multi-Agent。

## 三、Tool Calling

### 14.【基础】什么是 Tool Calling？

请完整描述从用户提问到模型生成最终答案的工具调用流程。

### 15.【基础】模型会真正执行函数吗？

请说明模型、应用程序和真实工具三者的责任边界。

### 16.【基础】一个工具定义通常包含哪些内容？

请说明工具名称、描述、参数 Schema 分别会怎样影响模型调用工具。

### 17.【场景】设计销售查询工具

请为“按月份和地区查询销售额”设计工具名称、工具描述、必填参数和返回结果。

### 18.【场景】为什么不能直接执行模型生成的参数？

假设模型生成了下面的调用参数，你会进行哪些校验？

```json
{
  "month": "下个月",
  "region": "全部地区"
}
```

### 19.【场景】工具执行失败如何处理？

如果数据库查询工具发生超时，应用程序应该向模型返回什么信息？Agent 应该如何决定重试、降级还是向用户报错？

### 20.【进阶】查询工具和写操作工具有什么不同？

请比较“查询订单”和“执行退款”的风险，并说明退款工具为什么需要权限校验、幂等控制和 Human-in-the-loop。

### 21.【进阶】Tool Calling 等于 Agent 吗？

请分别举出一个“使用 Tool Calling 但不是 Agent”和一个“通过 Tool Calling 构成 Agent 循环”的例子。

## 四、Memory

### 22.【基础】什么是 Agent Memory？

为什么说 Memory 是应用程序提供的能力，而不是大模型天然拥有的能力？

### 23.【基础】短期记忆和长期记忆有什么区别？

请从生命周期、保存内容和存储方式三个角度比较。

### 24.【基础】短期记忆和上下文窗口有什么区别？

如果已经保存了 100 轮对话，为什么不应该在每次请求中把它们全部发给模型？

### 25.【基础】长期记忆为什么经常使用向量数据库？

向量数据库适合保存和检索什么信息？哪些信息更适合存入关系数据库或 KV 数据库？

### 26.【基础】Memory 和 RAG 有什么区别？

请判断下面的信息分别更适合放入 Memory 还是 RAG：

```text
A. 公司销售指标口径说明书
B. 用户偏好金额使用万元
C. 产品操作手册
D. 用户上次分析华东区销售额的结论
```

### 27.【场景】长期记忆应该保存什么？

用户在一次对话中说了下面四句话，哪些值得保存为长期记忆？为什么？

```text
A. 谢谢
B. 以后经营报告金额都用万元，并保留两位小数
C. 我今天有点困
D. 我负责华东区经营分析
```

### 28.【场景】如何处理记忆冲突？

用户先说“金额默认使用元”，后来又说“以后统一使用万元”。系统应该如何更新、保留或删除旧记忆？

### 29.【进阶】Memory 有哪些安全风险？

请至少从用户隔离、敏感信息、记忆删除、错误记忆和 Prompt Injection 五个方面回答。

## 五、综合场景题

### 30.【综合】设计一个支持记忆的 BI Agent

用户说：

```text
按照我平时的口径，对比上个月和这个月的销售情况，找出下降原因。
```

请说明：

- Agent 如何理解“平时的口径”
- 需要读取哪些短期和长期记忆
- 应该使用哪些工具
- 适合使用哪种架构模式
- 完整执行流程是什么

### 31.【综合】Agent 为什么反复调用同一个工具？

一个 ReAct Agent 连续五次使用相同参数调用销售查询工具。请从工具描述、状态管理、终止条件、错误返回和模型行为几个方面分析原因，并提出改进方案。

### 32.【综合】如何保证经营报告可信？

一个 Agent 自动查询数据库并生成经营分析报告。请设计一套机制，降低数据错误、模型幻觉、错误工具调用和结论不可追溯的问题。

### 33.【综合】什么时候需要人工介入？

BI Agent 根据分析结果准备向管理层发送预警邮件。请说明哪些步骤可以自动完成，在哪个节点加入 Human-in-the-loop，以及人工需要确认什么。
